= Overview

Neo4j Code Graph is change-risk and vulnerability-reachability intelligence for Java codebases. It turns a repository and its Git history into a queryable Neo4j graph, then answers the question dependency scanners can't: given a CVE in a dependency, which of your methods can actually reach it, how big is the blast radius of fixing it, and who owns that code? It joins four signals no open-source tool combines in one place — git history as a graph, versioned Maven→CVE linkage, a method-level call graph (including calls into external library APIs), and GDS analytics. It is honest, ranked triage on a lexical call graph, not a sound static analyser; for proof-grade dataflow use CodeQL or Joern.

image::overview-art.jpg[Overview art,align=center,width=80%]

What it solves:

- *Security and risk visibility*: surface vulnerable third‑party dependencies and where they intersect with public APIs or recent changes
- *Architecture and technical debt*: highlight hot files/methods by size/complexity and graph centrality to prioritize refactors
- *Temporal coupling and release risk*: identify files that frequently change together and recent change clusters
- *Team insights*: find subject‑matter experts by commit history

Key capabilities:

- *End‑to‑end pipeline with Prefect*: code analysis, Git history ingestion, centrality, CVE analysis, temporal coupling
- *Aura‑compatible analytics*: APOC core and standard GDS only
- *Batteries‑included querying*: catalog of Cypher patterns for coupling, risk, refactors, expert finding, executive summaries
- *Docs and guardrails*: CI builds docs, validates Cypher snippets against an ephemeral Neo4j, and publishes to GitHub Pages
